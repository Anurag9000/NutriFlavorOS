## üì• 1. Installation

### Backend Setup
```bash
# Clone the repository
git clone https://github.com/your-username/NutriFlavorOS.git
cd NutriFlavorOS

# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies (Core + ML)
pip install -r requirements.txt
```

### Frontend Setup
```bash
cd frontend
npm install
cd ..
```

---

## üîë 2. Configuration (Environment Variables)

Create a `.env` file in the `backend/` directory with your API keys:

```ini
# backend/.env

# --- Scientific Databases ---
RECIPEDB_BASE_URL=https://api.cosylab.iiitd.edu.in/recipedb
RECIPEDB_API_KEY=YOUR_KEY_HERE

FLAVORDB_BASE_URL=https://cosylab.iiitd.edu.in/flavordb
FLAVORDB_API_KEY=YOUR_KEY_HERE

SUSTAINABLEFOODDB_BASE_URL=https://api.sustainablefooddb.com
SUSTAINABLEFOODDB_API_KEY=YOUR_KEY_HERE

DIETRXDB_BASE_URL=https://api.dietrxdb.com
DIETRXDB_API_KEY=YOUR_KEY_HERE

# --- ML Configuration ---
MODEL_WEIGHTS_PATH=./ml/weights
DeepTastePredictor_VERSION=v2.1
```

## üß† 3. Train ML Models (The "AI" Part)

Before running the server, initialize and "train" the ML models to ensure they have baseline weights.

```bash
# Run the training simulation pipeline
python -m backend.scripts.train_demo_models
```
*Output should show losses decreasing and "Models Saved Successfully".*

---

## üöÄ 4. Running the Platform

### Start Backend Server (API)
```bash
# From root directory
cd backend
uvicorn main:app --reload --port 8000
```
*API will differ at `http://localhost:8000/docs`*

### Start Frontend (UI)
```bash
# Open a new terminal
cd frontend
npm start
```
*App will launch at `http://localhost:3000`*
